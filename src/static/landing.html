How it works
Three steps, zero guesswork.
Shadow Developer runs as a parallel audit layer alongside your AI coding agent.
01 — INTERCEPT

Prompt captured

Every feature prompt is intercepted before code generation begins. Shadow Developer pulls your project's architectural context from memory.

02 — AUDIT

Adversarial analysis

Two agents run in parallel: one drafts the spec, the other generates an adversarial spec. Security rules, API drift checks, and OWASP analysis run against both.

03 — VERDICT

Block or approve

Critical issues halt code generation entirely. High-severity findings require revision. Clean prompts proceed with auto-generated integration tests.

Detection Coverage
What gets caught.
Rules run against both the feature prompt and scanned source files in real time.
SQL Injection
OWASP A03

Detects raw f-string queries and cursor.execute with string interpolation.

Auth Bypass
OWASP A01

Flags prompts and code that skip or disable authentication and authorization checks.

Hardcoded Secrets
OWASP A02

API keys, passwords, tokens, and private keys assigned directly in source.

Webhook Signature
SEC

Webhook routes that lack HMAC-SHA256 signature verification are blocked.

API Route Collision
DRIFT

New routes that conflict with existing registered paths and HTTP methods.

Schema Conflict
DRIFT

Model changes that will require database migrations or break existing queries.

Weak Cryptography
OWASP A02

MD5, SHA-1, and non-cryptographic random in security-sensitive contexts.

Dependency Conflict
BUILD

Version mismatches between requested libraries and pinned project dependencies.

Shell Injection
OWASP A03

subprocess calls with shell=True and os.system with user-controlled input.

Insecure Deserialization
OWASP A08

pickle.loads and yaml.load without Loader on untrusted data sources.

SSL Verification Off
OWASP A07

requests and httpx calls with verify=False disable certificate validation.

Rate Limiting Missing
SEC

New public endpoints without mention of rate limiting or throttling logic.
